Behavior Options

Customize drawer behavior with various interaction options.

🎛️ Drawer Behavior Control Properties

Control how your drawer behaves with these input properties. Each property has a specific purpose and use case.

showOverlay
boolean = true

Controls whether to show the dark background overlay behind the drawer.

[showOverlay]="true" - Standard modal behavior
[showOverlay]="false" - Non-modal sidebar
closeOnOverlayClick
boolean = true

Controls whether clicking the overlay closes the drawer.

[closeOnOverlayClick]="true" - Easy dismissal
[closeOnOverlayClick]="false" - Prevent accidents
closeOnEscape
boolean = true

Controls whether pressing the Escape key closes the drawer.

[closeOnEscape]="true" - Keyboard accessibility
[closeOnEscape]="false" - Critical workflows
showCloseButton
boolean = true

Controls whether to show the X close button in the header.

[showCloseButton]="true" - Clear exit option
[showCloseButton]="false" - Custom triggers
persistent
boolean = false

Makes the drawer "sticky" - cannot be closed by user actions.

[persistent]="false" - Normal behavior
[persistent]="true" - Force interaction
animate
boolean = true

Controls whether to use spring animation when opening/closing.

[animate]="true" - Smooth transitions
[animate]="false" - Performance mode

✅ Overlay Enabled

[showOverlay]="true"

Notice the dark background overlay that dims the page content. This is the standard modal behavior that focuses attention on the drawer.

  • ✅ Background is dimmed
  • ✅ Page content is not interactive
  • ✅ Focus is trapped in drawer

❌ Overlay Disabled

[showOverlay]="false"

No background overlay - the page content remains visible and interactive. Perfect for non-modal sidebars and panels.

  • ✅ Background remains visible
  • ✅ Page content stays interactive
  • ✅ Great for navigation panels

👆 Click Overlay to Close

[closeOnOverlayClick]="true"

Try clicking the dark background area to close this drawer. This is standard modal behavior for easy dismissal.

  • ✅ Quick and intuitive closing
  • ✅ Standard user expectation
  • ✅ Good for temporary content

🚫 Overlay Click Disabled

[closeOnOverlayClick]="false"

Clicking the overlay won't close this drawer. Use the close button or ESC key instead.

  • ✅ Prevents accidental closing
  • ✅ Good for important forms
  • ✅ Forces intentional interaction

⌨️ Press ESC to Close

[closeOnEscape]="true"

Try pressing the Escape key to close this drawer. This is standard accessibility behavior for keyboard users.

  • ✅ Keyboard accessibility
  • ✅ Quick escape route
  • ✅ Standard web behavior

🔒 ESC Key Ignored

[closeOnEscape]="false"

The Escape key won't close this drawer. Use the close button instead.

  • ✅ Prevents accidental closing
  • ✅ Good for critical workflows
  • ✅ Forces completion

❌ Close Button Visible

[showCloseButton]="true"

Notice the X button in the top-right corner. This provides a clear way for users to close the drawer.

  • ✅ Clear visual exit
  • ✅ Familiar UI pattern
  • ✅ Always accessible

🚫 Close Button Hidden

[showCloseButton]="false"

No X button is shown in the header. Users must close via overlay click, ESC key, or custom buttons.

  • ✅ Clean header design
  • ✅ Custom close triggers
  • ✅ Guided workflows

✅ Normal Behavior

[persistent]="false"

This drawer can be closed by multiple methods:

  • ✅ Clicking the overlay
  • ✅ Pressing ESC key
  • ✅ Clicking the close button
  • ✅ Standard user-friendly behavior

🔒 Persistent Mode

[persistent]="true"

This drawer ignores user attempts to close it:

  • ❌ Overlay clicks are ignored
  • ❌ ESC key presses are ignored
  • ✅ Only close button works
  • ✅ Perfect for critical forms

🌸 Smart Animate Spring

[animate]="true"

Notice the dynamic spring animation when this drawer opened. This provides an engaging user experience.

  • ✅ Bouncy spring motion
  • ✅ Energetic feeling transitions
  • ✅ Enhanced user experience
  • ✅ Modern, lively polish

⚡ Instant Appearance

[animate]="false"

This drawer appeared instantly without animation. Better for performance-critical applications.

  • ✅ Instant response
  • ✅ Better performance
  • ✅ Reduced motion preference
  • ✅ Accessibility friendly
