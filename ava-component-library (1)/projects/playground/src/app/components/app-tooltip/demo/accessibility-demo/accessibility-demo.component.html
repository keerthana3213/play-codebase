Accessibility Checklist

✅ Keyboard accessible (focus trigger)
✅ Proper ARIA attributes (aria-describedby, role)
✅ Screen reader compatible
✅ Focus management maintained
✅ High contrast support
✅ Reduced motion support
✅ Color contrast compliance
✅ Touch-friendly on mobile

Best Practices

Do

  • Use focus trigger for form elements
  • Provide alternative ways to access information
  • Keep tooltip content concise
  • Test with screen readers
  • Support keyboard navigation
  • Respect user motion preferences

Don't

  • Put critical information only in tooltips
  • Use hover-only tooltips for important content
  • Make tooltips too long or complex
  • Ignore high contrast mode
  • Prevent focus from moving naturally
  • Force animations on users who prefer reduced motion
