Usage Examples

Basic Tree Structure

// Define your tree data
const treeData: TreeNode[] = [
  {
    name: 'Documents',
    isExpanded: false,
    children: [
      {
        name: 'Work',
        children: [
          { name: 'Resume.pdf' },
          { name: 'Cover Letter.doc' }
        ]
      }
    ]
  }
];
 
// Use in template
<aava-tree [data]="treeData"></aava-tree>

With Selection

// Component
selectedNodes: TreeNode[] = [];
 
onSelectionChange(selection: TreeNode[]): void {
  this.selectedNodes = selection;
  console.log('Selected:', selection);
}
 
// Template
<aava-tree
  [data]="treeData"
  selectionMode="single"
  [selectedNodes]="selectedNodes"
  (selectionChange)="onSelectionChange($event)">
</aava-tree>

With Icons and Events

// Tree data with icons
const treeData: TreeNode[] = [
  {
    id: 1,
    name: 'Projects',
    icon: 'folder',
    isExpanded: true,
    children: [
      {
        id: 2,
        name: 'Website',
        icon: 'globe'
      },
      {
        id: 3,
        name: 'Mobile App',
        icon: 'smartphone'
      }
    ]
  }
];
 
// Handle node clicks
onNodeClick(node: TreeNode): void {
  console.log('Clicked node:', node.name);
  // Custom logic here
}
 
// Template
<aava-tree
  [data]="treeData"
  (nodeClick)="onNodeClick($event)">
</aava-tree>

Best Practices

🎯 Use Unique IDs

Always provide unique id values for nodes when using selection functionality to ensure proper state management.

🎨 Consistent Icons

Use consistent icon naming from the Lucide icon set. Common patterns: folder for containers, file for items.

⚡ Performance

For large trees, consider implementing virtual scrolling or lazy loading of child nodes to maintain performance.

♿ Accessibility

The component includes built-in keyboard navigation. Ensure your tree data has meaningful names for screen readers.

🔄 State Management

Use controlled selection state with [selectedNodes] and (selectionChange) for predictable behavior.
