State Overview

Badge states define the color scheme and visual priority of the badge. Each state has specific use cases and semantic meaning.

High Priority

Used for critical alerts, errors, or urgent notifications that require immediate attention.

Medium Priority

Used for warnings, pending actions, or notifications that need attention but aren't critical.

Low Priority

Used for success states, completed actions, or informational content that's positive.

Neutral

Used for general information, default states, or content that doesn't have specific priority.

Information

Used for informational content, help text, or general updates that provide context.

Usage Guidelines

Best practices for choosing the right badge state for your use case.

High Priority

  • Critical system errors or failures
  • Security alerts or breaches
  • Urgent user notifications
  • Payment failures or timeouts

Medium Priority

  • Warning messages
  • Pending approvals or actions
  • System maintenance notices
  • Feature deprecation warnings

Low Priority

  • Successful operations
  • Completed tasks or processes
  • Positive status indicators
  • Confirmation messages

Neutral

  • General information
  • Default states
  • Non-critical updates
  • Background processes

Information

  • Help text and tips
  • Feature announcements
  • Educational content
  • Contextual information
